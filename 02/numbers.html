Введение

Множествам целых Z и действительных R чисел на компьютере соответствуют их машинные аналоги ZM и RM, причём свойства этих множеств существенно зависят от используемого языка программирования. Непонимание особенностей представления чисел в компьютере может иметь катастрофические последствия для программиста.

Целые числа в различных языках программирования

В большинстве языков программирования множество целых чисел является конечным: ZM⊂Z. Этот факт является следствием того, что для хранения любого целого числа используется фиксированный объём памяти (в настоящее время чаще всего 4 байта). Таким образом, существует как минимально представимое целое число, так и максимально представимое. Причём этот отрезок числовой оси как бы «свёрнут в кольцо», из-за чего следующим за максимальным целым числом оказывается самое минимальное:

Практически все языки предоставляют специальные библиотеки, с помощью которых можно работать с «настоящими целыми числами». Но для «целых чисел по умолчанию» (элементов множества ZM) оказываются не всегда выполнены следующие свойства:

  • ∀xy>0⇒x+y>x;
  • ∀x>0y>1⇒x⋅y>x.

Действительные числа

Для записи действительных чисел практически во всех языках программирования наряду с «обычной» формой используют ещё и экспоненциальную форму:

Запись Значение
1.23 1.23
-0.002 -0.002
1.0e+3 1000.0
1.0e3 1000.0
2.7E-2 0.027
-3.1e+2 -310.0

Язык Python позволяет не писать ноль после точки при записи действительного числа: вместо 1.0e6 можно написать 1.e6.

Во всех языка программирования множество «машинных» действительных чисел RM является конечным. Это — следствие того, что для хранения произвольного действительного числа используется фиксированное количество байт памяти. Чаще всего восемь байтов (см. Double-precision floating-point format). Так как множество R «настоящих действительных чисел» является не только бесконечным, но и полным (между любыми двумя различными числами найдётся третье, отличное от этих двух), то свойства «машинных» действительных чисел очень сильно отличаются от свойств «настоящих» действительных чисел.

>>> .1 + .1 + .1 == .3
False

Округление в языке Python по умолчанию является «банковским»:

>>> round(3.5)
4
>>> round(2.5)
2

Округление — совсем не такая простая операция, как это может показаться на первый взгляд. Желающие детально разобраться в различных стратегиях округления могут ознакомиться со статьёй Rounding в Википедии.

Важнейшие характеристики множества RM для конкретного компьютера можно узнать следующим образом:

>>> import sys
>>> sys.float_info
sys.float_info(max=1.7976931348623157e+308,
max_exp=1024, max_10_exp=308,
min=2.2250738585072014e-308, min_exp=-1021,
min_10_exp=-307, dig=15, mant_dig=53,
epsilon=2.220446049250313e-16, radix=2,
rounds=1)

Приведённый выше рисунок, на котором изображено множество неотрицательных элементов множества RM, объясняет, почему не для всех элементов этого множества верны следующие утверждения, всегда справедливые для элементов множества R:

  • 1+x>1 для x>0;
  • x+x существует;
  • (x⋅2)/2=x;
  • (x/2)⋅2=x;
  • a+(b+c)=(a+b)+c;
  • a+b+c=c+b+a.

Вот один из примеров, иллюстрирующих различие свойств множеств R и RM:

>>> 1.e-8 + 1.e-8 + 1.e-8 + 1.e-8 + 1.e-8 + 1.e-8 + 1.e9 > 1.e9
True
>>> 1.e9 + 1.e-8 + 1.e-8 + 1.e-8 + 1.e-8 + 1.e-8 + 1.e-8 > 1.e9
False
