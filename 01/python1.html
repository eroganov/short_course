Постановка задачи

Напишите программу на языке Python, читающую со стандартного ввода целое число n и печатающую на стандартный вывод сумму всех целых чисел между единицей и целым числом n включительно.

Пример 1

Ввод: 1

Вывод: 1

Пример 2

Ввод: 3

Вывод: 6

Программу, код которой размещён в файле с именем filename.py, можно выполнить, набрав в терминале следующую команду:

python filename.py

Стандартный ввод и стандартный вывод — потоки ввода-вывода в различных операционных системах, которые позволяют процессу, создаваемому во время интерпретации Python-кода, взаимодействовать с «внешним миром», читая (в случае стандартного ввода) или записывая (для стандартного вывода) какие-то данные. По умолчанию стандартный ввод ассоциирован с клавиатурой, а стандартный вывод — с терминалом, в котором выполнена команда запуска интерпретатора.

Очень легко перенаправить стандартный ввод и/или стандартный вывод. Так команда

python filename.py <data.txt >result.dat

будет читать данные не с клавиатуры, а из файла с именем data.txt, её вывод окажется в файле result.txt. Если второй из этих файлов существовал, то его предыдущее содержимое будет уничтожено. Для добавления информации в конец существующего файла вместо > следует использовать >>, например

python filename.py <data.txt >>result.dat

Обсуждение алгоритма решения

Каков алгоритм решения этой задачи?

Имея алгоритм, можно относительно легко записать его (закодировать) на любом известном языке программирования. Для этого, конечно, надо знать используемый язык…

Основная масса ошибок при решении различных задач заключается именно в некорректном алгоритме. Хотя ошибиться можно и на этапе кодирования (написания программы).

Алгоритм решения рассматриваемой задачи предполагает выполнение повторяющихся действий, что в различных языках программирования часто реализуется с помощью так называемых циклов. В данном случае к некоторой переменной, в которой будет накапливаться требуемая сумма, в цикле надо добавлять очередное число. Начать этот процесс можно с добавления единицы, завершить следует добавлением величины n, а добавляемую величину на каждом шаге увеличивать на один.

Запись алгоритма в виде программы

Вот как можно попытаться реализовать этот алгоритм на языке Python:

n = int(input())
s = 0
i = 1
while i <= n:
    s = s + i
    i = i + 1
print(s)

Даже человек, ранее никогда не слышавший о языке Python ничего, в состоянии понять почти всё в этой программе. Тем не менее здесь следует остановиться и тщательно обсудить каждый символ кода.

Оператор присваивания в языке Python, как и во многих других языках программирования, имеет вид имя_переменной = выражение.

Функция input() читает одну строку со стандартного ввода, а функция int() преобразует считанную строку в целое число (для получения действительного числа следует использовать функцию float()).

Условие i <= n цикла while проверяется перед началом выполнением цикла и после каждой его итерации. В случае его истинности выполняется тело цикла, которое отделяется от условия двоеточием. Строки тела цикла должны иметь одинаковый (положительный) отступ по сравнению со строкой начала цикла. Рекомендуется использовать отступ величиной в 4 позиции.

Функция print() печатает её аргумент на стандартный вывод.

Если написанную программу запустит человек, не знающий, что именно она делает, то у него может сложиться мнение, что программа «не работает», так как программа будет ждать ввода числа, а человек — какой-то активности от программы. Поэтому для интерактивных программ (предполагающих взаимодействие с пользователем) любому вводу обычно предшествует вывод текста-подсказки. Вывод результатов в них также обычно содержит какие-то пояснения.

Важно понимать, что программист не имеет права самостоятельно выбирать тот или иной стиль общения с пользователем. Всё определяет формулировка задача, которую часто называют спецификацией. Если поведение программы не соответствует её спецификации, то программа является неправильной!

Вот как может быть изменён вышеприведённый код для интерактивной программы:

n = int(input("Введите целое число -> "))
s = 0
i = 1
while i <= n:
    s = s + i
    i = i + 1
print(f"Сумма целых чисел между единицей и числом {n} включительно равна {s}")

Выполните эту программу несколько раз для различных значений n, например, 1, 2 и 3.

Замечание 1. В языке Python чрезвычайно важным является правильное форматирование кода программы. Если добавить в первую позицию первой строки файла с программой пробел, то при попытке запуска мы получим диагностическое сообщение IndentationError: unexpected indent — неожиданный отступ.

Большая часть современных редакторов автоматически предлагает правильные отступы, но догадаться, где должно завершиться тело цикла, не может ни один из них. Если последнюю строку программы сдвинуть вправо на четыре позиции, то программа останется синтаксически правильной, но её поведение изменится. Посмотрите, как именно.

Замечание 2. Для записи строк в языке Python можно использовать как кавычки ("), так и апострофы ('). Разница проявляется только в тех случаях, когда в саму строку должен быть включён один из этих символов.

Замечание 3. В последней строке программы аргументом функции print является так называемая f-строка — formatted string literal. Такие строки появились в языке Python только начиная с версии 3.6, а их достоинством является возможность подстановки в них значений переменных с помощью фигурных скобок, что очень удобно.

Важнейшие свойства программ

Правильность программы

Правильной называется программа, которая при всех допустимых в соответствии с постановкой задачи входных данных заканчивает работу за конечное время и выдаёт корректный результат.

Является ли написанная программа правильной?

Программа не является правильной!

Для того чтобы доказать это, достаточно привести пример таких входных данных, на которых написанная программа «не работает».

Можете это сделать?

Исправить ошибку можно многими разными способами. Предложенный ниже вариант использует отсутствующую во многих языках программирования возможность обмена значений двух переменных, реализуемую с помощью так называемого множественного присваивания:

n = int(input())
s = 0
i = 1
if n < i:
    i, n = n, i
while i <= n:
    s = s + i
    i = i + 1
print(s)

Временна́я сложность программы

Среди свойств, характеризующих алгоритмы (и программы, которые являются просто записью алгоритмов на том или ином языке программирования), правильность является важнейшим, но далеко не единственным показателем. Очень важной является временна́я сложность алгоритма, описывающая время его работы при различных входных данных.

Написанная нами программа имеет линейную сложность, так как количество выполняемых в ней операций (как и время работы) прямо пропорционально величине n. Не точно пропорционально, а лишь приблизительно. На языке математики это записывают так: Θ(n).

Попробуйте выполнить нашу программу при значении n, равным миллиарду!

Кто-то скажет: «язык Python слишком медленный». Действительно, интерпретатор этого языка работает иногда (хотя и далеко не всегда) на два порядка медленнее, чем программы, написанные на таких языках, как C, C++ или Java. А ведь n может равняться и триллиону и квадриллиону и квинтиллиону…

Использую более быстрый (компилируемый) язык можно добиться сокращения времени работы программы в 100 раз, но чаще всего этого бывает недостаточно. Проблема не в том, какой язык используется, а в том, какой алгоритм применяется!

При написании программы мы использовали слишком плохой (медленный) алгоритм!

Предложите алгоритм получше.

Подсказка: знание математики так часто помогает!

В данном случае достаточно вспомнить формулу для суммы арифметической прогрессии: полусумму первого и последнего её членов надо умножить на количество слагаемых.

n = int(input())
if n >= 1:
    k = n
else:
    k = abs(n) + 2
print((n + 1) * k / 2)

При запуске этой программы обнаруживается, что печатаемый ею результат не является целым числом. Это связано с тем, что в Python3 результат операции деления (/) всегда является числом действительным (float). Целочисленное деление обозначается двумя слэшами (//), поэтому программу лучше исправить:

n = int(input())
if n >= 1:
    k = n
else:
    k = abs(n) + 2
print((n + 1) * k // 2)

А как вывести формулу для суммы арифметической прогрессии?

Сложность новой программы константная (Θ(1)), что означает практическую неизменность времени её выполнения при любых значениях параметра n.

Для очень больших чисел увеличивается время выполнения любых операций над ними, поэтому время работы нашей последней программы всё же будет расти с ростом n.

Заметим, что вместо стандартной конструкции if-else в подобных случаях в других языках часто используют так называемый тернарный оператор ?:. В языке Python такого тернарного оператора нет, в нём его заменяют следующей конструкцией:

n = int(input())
k = n if n >= 1 else abs(n) + 2
print((n + 1) * k // 2)

Если попытаться дословно закодировать фразу «полусумму первого и последнего её членов надо умножить на количество слагаемых», то получится ошибочная программа

n = int(input())
k = n if n >= 1 else abs(n) + 2
print((n + 1) // 2 * k)

В чём причина её неправильности?

Программа может быть сделана ещё чуть-чуть более быстрой и простой:

n = int(input())
print((n + 1) * (1 + abs(n - 1)) // 2)
